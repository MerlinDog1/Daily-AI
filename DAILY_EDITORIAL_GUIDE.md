# Daily AI Editorial Guide (Baseline)

Use the 2026-03-10 digest style as the minimum quality bar.

## Required daily sections
1. Stats Grid (3 cards, quantified)
2. Breaking AI News (5 items)
3. Open Source AI (3 items)
4. Hot GitHub (6 items)
5. Run It Locally (3 bullets)
6. API Tools (3 bullets)
7. What We Should Try (3 concrete experiments)

## Coverage mix
- US labs/platforms
- China AI developments
- Open source + GitHub trends
- OpenClaw/workflow relevance
- Local/on-device AI and API tooling split

## Writing standard
- Rich summaries, not one-liners.
- Each item includes why-it-matters context.
- Keep tactical and actionable.
- End with implementation ideas for Liam’s active work.

## Source standard
- Prefer official/primary sources first.
- Add secondary media only as support.
- Keep links in every item.

## Ordering
- Newest entry always at top of `data/posts.json`.
